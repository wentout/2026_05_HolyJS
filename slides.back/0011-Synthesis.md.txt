# Перспектива TypeScript

<div className='codesmall'>

```typescript
// type — для данных (Type)
type UserData = {
    name: string;
    age: number;
};

// interface — для контрактов (Interface)
interface Constructable {
    new (): Instance;  // ← конструктор!
}
interface Callable {
    (): void;  // ← функция!
}

// Function — и то, и другое одновременно!
interface MyFunction extends Callable, Constructable {}
```

</div>

<div className="slide-callout">
<h3 className="secondary">type для структуры, interface для поведения</h3>
</div>

-----

# в JavaScript runtime граница размыта

<div className='codesmall'>

```javascript
// Function — это и Type и Interface одновременно
function myFunc() {}


// Как Type (объект):
myFunc.name;           // "myFunc"
myFunc.prototype;      // { constructor: ... }

// Как Interface (поведение):
myFunc();              // callable
new myFunc();          // newable


// И ещё...
myFunc instanceof Function;  // true
myFunc instanceof Object;    // true
```

</div>

<div className="slide-callout">
<h2 className="primary">Разделяют ли их в движке ?</h2>
</div>
