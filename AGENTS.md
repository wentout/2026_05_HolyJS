## Critical: gen.sh Auto-Generates list.txt

**Never manually edit `slides/list.txt`** - it is auto-generated by `../gen.sh`.

### gen.sh Script Logic

```bash
# 1. Remove old list.txt
rm slider/slides/list.txt

# 2. Find all slide files
names=$(ls ./slider/slides/ | grep -E '\.(json|md|mdx|txt)')

# 3. Count separators in each file
for t in $names; do
    # Count ----- for MDX or },{ for JSON arrays
    count=$(grep -E '(-----|\},\{)' -- $name | wc -l)
    echo $t $count >> ./slider/slides/list.txt
done
```

### Separator Patterns Counted

| File Type | Separator | Count = |
|-----------|-----------|---------|
| `.md.txt` | `-----` | number of `-----` lines |
| `.json` (array) | `},{` | array length - 1 |
| `.json` (object) | (none) | 0 |

**After creating/modifying slides, always run:**
```bash
cd ..
./gen.sh
cd slider
```

## Quick Reference: File Structure

```
slides/
├── list.txt              # AUTO-GENERATED - DO NOT EDIT
├── 0000.json             # Title/metadata slides
├── 0001.md.txt           # Content slides (MDX)
└── 0002.json             # More metadata slides
```

### list.txt Format

```
<filename> <sub-slide-count>
```

**Rules:**
- One entry per line
- Filename + space + number
- Files are loaded sequentially
- Number = count of sub-slides (1+ for JSON, N for MDX with `-----` separators)

**Example:**
```
0000.json 2       # 2 slides defined in JSON array
0001.md.txt 5     # MDX with 4 "-----" separators = 5 sub-slides
0002.json 1       # Single JSON object (count = 0 separators found, but minimum 1)
```

## Code Example Runner (exampleRunner.js)

Runtime code execution for interactive demonstrations:

```javascript
// src/exampleRunner.js
window.runExample = async function (path) {
  const fetched = await fetch(`../<project>/examples/${path}.js`, {
    headers: { "Content-Type": "text/plain" }
  });
  const src = await fetched.text();
  eval(src);
};
```

### Usage Flow

```
1. User clicks button in MDX slide
2. Button calls runExample('filename') 
3. Fetches public/examples/filename.js
4. Executes with eval()
5. Code runs in browser context
```

### File Locations

| Purpose | Path |
|---------|------|
| Example files | `public/examples/*.js` |
| Fetch URL | `../<project-name>/examples/{path}.js` |

**Note:** The project name in the fetch URL is hardcoded in `exampleRunner.js`. Update it when copying to a new project.

### Example File Format

```javascript

// Output goes to browser console
```

## Common Tasks

### Task: Add a New Slide

1. Check current `list.txt` to find the next number:
   ```bash
   tail -1 slides/list.txt
   # Output: 0010.md.txt 5
   ```

2. Create new file with next number:
   - If title/metadata: `0011.json`
   - If content: `0011.md.txt`

3. Add content to the file

4. **Regenerate list.txt**:
   ```bash
   cd ..
   ./gen.sh
   cd slider
   ```

### Task: Insert Slide in Middle

1. Create new slide file with appropriate number
2. **Regenerate list.txt**:
   ```bash
   cd ..
   ./gen.sh
   ```

The script automatically picks up all files in `slides/` directory.

### Task: Create Multi-Slide MDX

Use `-----` (5 dashes) to separate sub-slides:

```mdx
# Slide 1

Content

-----

# Slide 2

More content

-----

# Slide 3

Even more
```

**After saving, run `../gen.sh` to update the count in `list.txt`.**

### Task: Add Image

1. Place image in `public/content/` or `public/content/img/`
2. Reference in MDX:
   ```mdx
   <img src="./content/my-image.png" style={{width: '50vw'}} />
   ```
3. **Run `../gen.sh`** (if new files added)

### Task: Add Code Example

1. Place JS file in `public/examples/`
2. Add button in MDX:
   ```mdx
   <Button onClick={() => runExample('my-code')}>
     Run Example
   </Button>
   ```

3. **Run `../gen.sh`** (if new files added)

### Task: Create Custom View Component

1. Create component in `src/components/MyView.jsx`:
   ```jsx
   import React from 'react';
   
   const MyView = function () {
     Object.assign(this, this.slides.current);
   };
   
   MyView.prototype.View = function () {
     return (
       <div className="SlideContent">
         <h1>{this.title}</h1>
         <p>{this.customProp}</p>
       </div>
     );
   };
   
   export default MyView;
   ```

2. Register in `src/main.jsx`:
   ```jsx
   import MyView from './components/MyView';
   // ...
   SlideRoot.define(MyView);
   ```

3. Use in JSON:
   ```json
   {
     "view": "MyView",
     "title": "Custom",
     "customProp": "value"
   }
   ```

4. **Run `../gen.sh`**.

### Task: Add Footer to Custom View

## Debugging Tips

1. **Slides not appearing:** Run `../gen.sh` to regenerate `list.txt`
2. **list.txt changes lost:** It's auto-generated - edit slide files instead
3. **MDX not rendering:** Check file extension is `.md.txt` (not `.mdx`)
4. **Images not loading:** Use `./content/` prefix, not `/content/`
5. **Example not running:** Check path in `exampleRunner.js` fetch URL
6. **Build fails:** Run `npm run lint` to check for syntax errors
7. **Navigation stuck:** Press `Ctrl + Esc` to clear error state
8. **Component edits have no effect:** The build uses `.jsx` files, not `.tsx`. Check `src/components/index.js` to see which file is imported. If both `.jsx` and `.tsx` exist, the `.jsx` file is the one that matters.
