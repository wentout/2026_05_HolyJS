# const - это про привязку, не про значение

<div className='codesmall'>

```javascript
let toBeProp = 5;

const myObject = {};
Object.defineProperty(myObject, 'prop', {
    get () {
        return {
            [Symbol.toPrimitive] () {  // ← KEY: делает getter ленивым!
                return toBeProp;       //     возвращает ТЕКУЩЕЕ значение
            }
        };
    }
});

const { prop: prop1 } = myObject;  // prop1 — НЕ число!
                                   // это объект с Symbol.toPrimitive
```

</div>

<div className="slide-callout">
<h3 className="secondary">Getter + Symbol.toPrimitive = ленивое вычисление</h3>
<h3>const хранит ССЫЛКУ, не значение. Getter — это Interface.</h3>
</div>

# const переменная изменила значение!

<div className='codesmall'>

```javascript
let toBeProp = 5;
const myObject = {};
Object.defineProperty(myObject, 'prop', {
    get () {
        return { [Symbol.toPrimitive] () { return toBeProp; } };
    }
});
const { prop: prop1 } = myObject;

// Сюрприз: "const" переменная изменила своё значение:
console.log(prop1 + 0);   // 5
toBeProp = 7;             // изменили источник
console.log(prop1 + 0);   // 7 ?!?!?!?
```

</div>
