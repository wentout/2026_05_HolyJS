# Структурная (type) vs Номинальная (interface)

<div className='codesmall'>

```typescript
// Структурная: одинаковая структура = одинаковый тип
type Point  = { x: number; y: number };
type Coord  = { x: number; y: number };
// Point ≡ Coord  ✓  (структурно эквивалентны)

// Номинальная: разные имена = разные типы
interface IPoint { x: number; y: number; }
interface ICoord { x: number; y: number; }
// IPoint ≢ ICoord  ✗  (номинально различны)
```

</div>

<div className="slide-callout">
<h3 className="secondary">type — математика множеств. interface — имена классов.</h3>
</div>

-----

# Intersection: наивный подход не работает

<div className='codesmall'>

```typescript
// Наивное пересечение по перекрывающимся ключам:
type A = { x: 1 };
type B = { x: 2 };
type AB = A & B;  // { x: never }  ← КОНФЛИКТ!

// TypeScript intersection требует оба значения сразу.
// Но цепочки прототипов не сливаются — они ПЕРЕОПРЕДЕЛЯЮТ.
```

</div>

<div className="slide-callout">
<h3 className="danger">A & B on overlapping keys gives never</h3>
<h3>This is NOT how prototype inheritance works</h3>
</div>

-----

# Паттерн Proto: Pick + Exclude

<div className='codesmall'>

```typescript
// Корректно моделирует наследование цепочки прототипов:
export type Proto<P extends object, T extends object> =
    T & Pick<P, Exclude<keyof P, keyof T>>;

// Родитель: x=1, Потомок переопределяет на x=2
type Parent = { x: 1; y: 'a' };
type Child  = { x: 2 };
type Result = Proto<Parent, Child>;
// Result = { x: 2 } & Pick<Parent, 'y'>
// Result = { x: 2; y: 'a' }  ← Потомок ПОБЕЖДАЕТ!
```

</div>

<div className="slide-callout">
<h3 className="accent">Child properties take precedence over Parent</h3>
<h3>Same as JS: instance.x finds the closest prototype first</h3>
</div>

-----

# Почему interface НЕ МОЖЕТ это выразить

<div className='codesmall'>

```typescript
// Interface требует совместимости:
interface Child extends Parent {
    x: 2;  // Ошибка: x уже объявлен в Parent
}

// Но type с паттерном Proto МОЖЕТ:
type Child = Proto<Parent, { x: 2 }>;  // ✓  { x: 2; ... }

// interface extends = номинальная проверка совместимости
// type & Pick/Exclude = структурное переопределение
```

</div>

<div className="slide-callout">
<h3 className="secondary">interface extends forces the child to be compatible with parent</h3>
<h3 className="primary">But prototype chains are about OVERRIDING, not compatibility</h3>
</div>

-----

# Суть: type & моделирует цепочки прототипов

<div className='codesmall'>

```typescript
// Алгебра типов TypeScript:
type Instance = Proto<Base, Proto<Extension, Override>>;
// Результат: свойства из Override, недостающие из Extension, затем Base

// JavaScript runtime:
const instance = Object.setPrototypeOf(
    Override,
    Object.setPrototypeOf(Extension, Base)
);
instance.x;  // ближайшее в цепочке побеждает — тот же принцип!
```

</div>

<div className="slide-callout">
<h2 className="primary">type with & is the ONLY way to express "inherit with overrides"</h2>
<h3>interface with extends cannot — it forces structural compatibility</h3>
</div>
