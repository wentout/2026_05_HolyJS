# TypeScript perspective

<div className='codesmall'>

```typescript
// type — для данных (Type)
type UserData = &#123;
    name: string;
    age: number;
&#125;;

// interface — для контрактов (Interface)
interface Constructable &#123;
    new (): Instance;  // ← конструктор!
&#125;

interface Callable &#123;
    (): void;  // ← функция!
&#125;

// Function — и то, и другое одновременно!
interface MyFunction extends Callable, Constructable &#123;&#125;
```

</div>

<center>
<h3 style={{color: 'orange'}}>type для структуры, interface для поведения</h3>

# JavaScript runtime: граница размыта

<div className='codesmall'>

```javascript
// Function — и Type, и Interface одновременно
function myFunc() &#123;&#125;

// Как Type (объект):
myFunc.name;           // "myFunc"
myFunc.prototype;      // &#123; constructor: ... &#125;

// Как Interface (поведение):
myFunc();              // callable
new myFunc();          // newable

// И ещё...
myFunc instanceof Function;  // true
myFunc instanceof Object;    // true
```

</div>

<Box sx={{marginTop: '5vh'}}>
<center>
<h2 style={{color: 'cyan'}}>В JavaScript движке они были разделены?</h2>
<h2 style={{color: 'orange'}}>НЕТ. Они всегда были едины.</h2>
</center>
</Box>
