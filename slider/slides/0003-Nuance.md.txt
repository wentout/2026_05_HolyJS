# TS docs: &laquo;стилистический выбор&raquo;

<div className='codemedium'>

```typescript
// В TS оба варианта почти эквивалентны:
interface Point { x: number; y: number; }
type Point = { x: number; y: number; };

// Интерфейс может описывать функцию:
interface MyFn { (): void; }

// Type может описывать функцию:
type MyFn = () => void;
```

</div>

<div className="slide-callout">
<h3>В TypeScript компиляторе — да, разница минимальна.</h3>
<h3 className="secondary">Но в JavaScript runtime — разница ФУНДАМЕНТАЛЬНА.</h3>
</div>

-----

<div style={{marginTop: '8vh', marginLeft: '20%'}}>
<h2 className="secondary">Интерфейс — это контракт &laquo;До и После&raquo;</h2>
<h3> ↑ &nbsp; "До" — что нужно для создания (new (), аргументы)</h3>
<h3> ↑ &nbsp; "После" — что получится в результате</h3>
<h3> ↑ &nbsp; Оба эти описания — это Тип. Интерфейс их соединяет.</h3>
</div>

<div style={{marginTop: '7vh', marginLeft: '20%'}}>
<h2 className="accent">Тип — это ответ на вопрос &laquo;ЧТО?&raquo;</h2>
<h3> ↓ &nbsp; Какая структура? Какие поля? Какой тип значения?</h3>
<h3> ↓ &nbsp; Тип описывает данные, не поведение.</h3>
</div>

# Пример различия

<div className='codemedium'>
```typescript
    
// Interface содержит в себе и "до" и "после":
        interface Factory {
            // " до " : с какой сущностью
                new (config: Config): Instance;
            // " после " : что можем делать
                create (): Product;
        }
// Type — просто описание структуры сущности:
        type Config = {
                name: string;
                count: number;
        };
```
</div>
