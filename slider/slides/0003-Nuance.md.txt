# TypeScript docs говорят: "это стилистический выбор"

<div className='codesmall'>

```typescript
// В TS оба варианта почти эквивалентны:
interface Point { x: number; y: number; }
type Point = { x: number; y: number; };

// Интерфейс может описывать функцию:
interface MyFn { (): void; }

// Type может описывать функцию:
type MyFn = () => void;
```

</div>

<div className="slide-callout">
<h3>В TypeScript компиляторе — да, разница минимальна.</h3>
<h3 className="secondary">Но в JavaScript runtime — разница ФУНДАМЕНТАЛЬНА.</h3>
</div>

-----

<div className="slide-text">
<h2 className="secondary">Интерфейс — это контракт "до и после"</h2>
<h3>"До" — что нужно для создания (new (), аргументы)</h3>
<h3>"После" — что получится в результате</h3>
<h3>Оба эти описания — это Тип. Интерфейс их соединяет.</h3>
</div>

<div className="slide-text" style={{marginTop: 'var(--space-xl)'}}>
<h2 className="accent">Тип — это ответ на вопрос "ЧТО?"</h2>
<h3>Какая структура? Какие поля? Какой тип значения?</h3>
<h3>Тип описывает данные, не поведение.</h3>
</div>

-----

# Пример различия

<div className='codesmall'>

```typescript
// Interface с "до" и "после":
interface Factory {
    // "до": что нужно
    new (config: Config): Instance;
    
    // "после": методы экземпляра
    create(): Product;
}

// Type — просто описание структуры:
type Config = {
    name: string;
    count: number;
};
```

</div>

<div className="slide-callout">
<h3 className="primary">Interface — это алгоритм. Type — это данные.</h3>
<h3>Теперь посмотрим, как это работает в JavaScript...</h3>
</div>
