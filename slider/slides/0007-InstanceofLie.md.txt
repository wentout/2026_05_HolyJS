# Как работает instanceof?

<div className='codemedium'>

```javascript
function foo() { };
const bar = { a: 'a' };

// Создаём фейковую цепочку прототипов:
Object.setPrototypeOf(foo.prototype, bar);
const baz = Object.create(foo.prototype);

// Проверим:
console.log(baz instanceof foo);  // true
// Но baz НИКОГДА не создавался через new foo()!
```

</div>

<div className="slide-callout">
<h3>Цепочка: baz → foo.prototype → bar</h3>
</div>

# итого: instanceof &mdash; это traversal algorithm

```mermaid
graph LR
    A[baz] -->|__proto__| B[foo.prototype]
    B -->|__proto__| C[bar]
    C -->|__proto__| D[Object.prototype]
    D -->|__proto__| E[null]

    B -.->|проверка: .constructor === foo ?| F{совпало!}
    F -->|true| G[instanceof foo === true]
```

<div className="slide-callout">
<h3 className="secondary">instanceof идёт по цепочке __proto__ и ищет совпадение .constructor</h3>
<h3 className="secondary">если находит — возвращает true,<br/> даже если объект создавался не через new</h3>
<h3 className="primary">и таким образом instanceof проверяет не факт создания,<br/> а лишь структуру цепочки прототипов</h3>
</div>

# переопределяем instanceof

<div className='codemedium'>

```javascript
// пример создания для функционального конструктора
const MyFunctionCtor = function () {
    // мы в конструкторе, поэтому тут мы
    /* ... оставляем закладку ... */
    /* ... известную только нам  ... */
}

MyFunctionCtr [ Symbol.hasInstance ] = ( instance ) => {
    // мы в проверке instanceof, поэтому
    /* ... проверяем закладку ... */
    /* ... известную только нам  ... */
}

```

</div>

# переопределяем instanceof

class MyClassCtor { // и для классового
