# Mnemonica Slider Framework

A React-based presentation framework built on [mnemonica](https://www.npmjs.com/package/mnemonica) for prototype-based composition, with MDX support for rich slide content, Theme UI for styling, and Vite for fast development.

## Overview

The Mnemonica Slider Framework is a specialized presentation tool designed for technical talks and demonstrations. It combines the power of React with a unique prototype-based architecture (via mnemonica) to create flexible, component-driven slides.

## Technology Stack

| Technology | Purpose |
|------------|---------|
| **[mnemonica](https://www.npmjs.com/package/mnemonica)** | Prototype-based object composition and inheritance |
| **React 18** | UI component library with hooks and concurrent features |
| **Theme UI** | Consistent design system with sx prop styling |
| **MDX (@mdx-js/mdx)** | Markdown with JSX for slide content |
| **Vite** | Fast development server and optimized builds |
| **D3** | Data visualization for charts (pie, tree) |
| **magic-snowflakes** | Visual effects (optional snowflake animations) |

## Build System: gen.sh

The `../gen.sh` script in the parent directory automatically generates `slides/list.txt` by counting sub-slides in each file.

**What gen.sh does:**
1. Removes the old `list.txt`
2. Finds all `.json`, `.md`, `.mdx`, and `.txt` files in `slides/`
3. Counts sub-slide separators (`-----` for MDX, `},{` for JSON arrays)
4. Outputs `filename count` to `list.txt`

**Never edit `list.txt` manually** - it will be overwritten on the next build.

## How to Create Slides

Slides are defined in the `slides/` directory:
1. **list.txt** - Auto-generated manifest (run `../gen.sh` to regenerate)
2. **Slide files** - Individual slide content (JSON or MDX)

### The list.txt Format (Auto-Generated)

```
0000.json 2       # JSON file with 2 sub-slides
0000.md.txt 1     # MDX file with 1 sub-slide  
0001.json 1       # Title slide with 1 sub-slide
0002.md.txt 3     # Content slide with 3 sub-slides
```

Format: `filename number` where:
- **filename**: The slide file (`.json` or `.md.txt`)
- **number**: How many sub-slides (separated by `-----` in MDX files, or `},{` in JSON arrays)

### JSON Slide Files

JSON files define structured slides with specific views:

```json
{
  "view": "Title",
  "title": "My Presentation",
  "subtitle": "A Great Talk"
}
```

**Available JSON views:**
- `"Title"` - Main title slide with title and subtitle
- `"Starter"` - Introduction slide with title and author
- `"Title2"` - Alternative title layout

**JSON Properties:**
| Property | Type | Description |
|----------|------|-------------|
| `view` | string | Component type to render |
| `title` | string | Main heading |
| `subtitle` | string | Subheading |
| `author` | string | Presenter name |
| `snowflakes` | boolean | Enable snow effect |

**Multiple slides in one JSON file:**

```json
[
  {
    "view": "Title",
    "title": "Slide 1",
    "subtitle": "First"
  },
  {
    "view": "Title",
    "title": "Slide 2", 
    "subtitle": "Second"
  }
]
```

### MDX Slide Files (.md.txt)

MDX files allow rich content with React components inside Markdown. Separate sub-slides with `-----`.

```mdx
# First Sub-Slide

Some content here.

<Box sx={{textAlign: 'center'}}>
  <img src="./content/my-image.png" style={{width: '50vw'}} />
</Box>

-----

# Second Sub-Slide

- Point 1
- Point 2

-----

# Code Example

```jsx
const Component = () => <div>Hello</div>;
```
```

**Default view for .md.txt files:** `"MDX"`

## Component Hierarchy

The framework uses a nested prototype hierarchy via mnemonica:

```
Runner (Main)
└── SlideRoot (Slide)
    ├── Title
    ├── Title2
    └── SlideStarter (Starter)
        ├── Progressor
        └── Footer
    └── SlideMDX (MDX)
        ├── Progressor
        └── Footer
```

### Available Components

| Component | Purpose | Props |
|-----------|---------|-------|
| `Slide` | Base slide container and renderer | - |
| `Title` | Title slide with h1 + subtitle | `title`, `subtitle` |
| `Title2` | Alternative title layout | `title`, `subtitle` |
| `Starter` | Introduction slide | `title`, `author` |
| `MDX` | Renders MDX content | `data` (mdx content) |
| `Footer` | Navigation footer | `count`, `index`, `level`, `among` |
| `Progressor` | Progress bar | `counters` |
| `Chart` | D3 visualization | `id`, `opts` |

### MDX Available Components

When writing MDX slides, these components are available:

| Component | Source | Usage |
|-----------|--------|-------|
| `Box` | theme-ui | Layout container with sx prop |
| `Heading` | theme-ui | Styled headings |
| `Button` | theme-ui | Interactive buttons |
| `Prism` | @theme-ui/prism | Syntax highlighting |
| `Chart` | custom | D3 charts (pie, tree) |
| `app` | custom | Access slide data via `{app slide.key}` |

### Styling with Theme UI

Use the `sx` prop for inline styles:

```mdx
<Box sx={{
  textAlign: 'center',
  marginTop: '25vh',
  color: 'teal'
}}>
  <h1>Centered Content</h1>
</Box>
```

**Theme UI spacing units:** Based on an 8px grid. `marginTop: '25vh'` uses viewport units directly.

## Running Code Examples

The framework includes a runtime code example loader (`src/exampleRunner.js`).

### How to Use in Slides

1. Place JavaScript files in `public/examples/`
2. Reference them in MDX with a button that calls `runExample()`:

```mdx
<Button onClick={() => runExample('01_1995')}>
  Run Example
</Button>
```

This will:
1. Fetch `public/examples/01_1995.js`
2. Execute the code with `eval()`
3. Output goes to `console.log`

### Example File Structure

```
public/examples/
├── 00_with.js
├── 01_1995.js
├── 02_01_SimpleClass.js
└── ...
```

### Example File Format

```javascript
// public/examples/my-example.js
const result = someFunction();
console.log(result);
```

**Note:** Examples run in the browser context via `eval()`. The `runExample()` function is globally available on `window`.

## Special Features

### Error Recovery
If an error occurs, the framework enters an "errored" state. Press `Ctrl + Esc` to continue.

### Progress Tracking
- Footer shows current position (e.g., `01.2 » 10`)
- Progress bar at the top indicates overall progress
- Sub-slides are tracked within slide groups

### Snowflakes Effect
Enable on any slide by adding `"snowflakes": true` to the JSON:

```json
{
  "view": "Title",
  "title": "Winter Talk",
  "snowflakes": true
}
```

### Charts
Embed D3 charts in MDX:

```mdx
<Chart id="my-chart" opts={{
  title: "Chart Title",
  chart: {
    type: "pie",
    data: [
      { label: "A", value: 30 },
      { label: "B", value: 70 }
    ]
  }
}} />
```

Chart types: `pie`, `tree`

### Accessing Slide Data

Use the `app` component to display slide properties:

```mdx
<p>Current slide title: {app slide.title}</p>
```

## Build and Run

### Development

```bash
# Run the generator and start dev server
npm run dev
```

This executes:
1. `../gen.sh` - Generates `slides/list.txt`
2. `vite` - Starts development server

The dev server runs on port 3030 by default.

### Build for Production

```bash
# Build optimized production bundle
npm run build
```

This executes:
1. `../gen.sh` - Regenerates `slides/list.txt`
2. `tsc` - TypeScript compilation
3. `vite build` - Production build
4. `cp -r ./slides ../docs/slides` - Copies slides to output

Output goes to `../docs` directory (configurable in `vite.config.ts`).

### Preview Production Build

```bash
# Preview the production build locally
npm run preview
```

Runs on port 8080.

### Print Mode

Append `#print` to the URL for print mode:
```
http://localhost:3030/#print
```

## Example: Creating a New Presentation

### Step 1: Create list.txt (via gen.sh)

Create your slide files first, then run:
```bash
cd ..
./gen.sh
```

This will auto-generate `slides/list.txt`.

### Step 2: Create Title Slide (0000.json)

```json
[{
  "view": "Title",
  "title": "My Awesome Talk",
  "subtitle": "HolyJS 2026"
},{
  "view": "Starter",
  "title": "About Me",
  "author": "Your Name"
}]
```

### Step 3: Create Content Slide (0001.md.txt)

```mdx
# Agenda

- Introduction
- Main Topic
- Conclusion

-----

# Introduction

Welcome to my talk!

<Box sx={{textAlign: 'center'}}>
  <img src="./content/my-photo.jpg" style={{width: '30vw'}} />
</Box>

-----

# Code Example

```javascript
function hello() {
  return "Hello World!";
}
```

<Button onClick={() => runExample('hello-example')}>
  Run Code
</Button>

-----

# Chart

<Chart id="chart1" opts={{
  title: "Data",
  chart: {
    type: "pie",
    data: [
      { label: "Yes", value: 75 },
      { label: "No", value: 25 }
    ]
  }
}} />

-----

# Thank You!

Questions?
```

### Step 4: Create Code Example (public/examples/hello-example.js)

```javascript
const greeting = "Hello from the example runner!";
console.log(greeting);
alert(greeting);
```

### Step 5: Create End Slide (0002.json)

```json
{
  "view": "Title",
  "title": "Thank You!",
  "subtitle": "Q & A"
}
```

### Step 6: Regenerate list.txt and Run

```bash
cd ..
./gen.sh
cd slider
npm run dev
```

Navigate to `http://localhost:3030`

export const NICK_TITLE = "Your Name";  // Footer name

```

## Tips

1. **Image paths**: Use `./content/` prefix for images in the `public/content/` folder
2. **Viewport units**: Use `vw` and `vh` for responsive sizing
3. **Colors**: Theme UI provides a dark preset. Use named colors or hex values
4. **Sub-slides**: Use `-----` in MDX files to create multiple sub-slides
5. **File naming**: Use leading zeros for ordering (0000, 0001, etc.)
6. **Regenerate list.txt**: Always run `../gen.sh` after adding/removing slides
